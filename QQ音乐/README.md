# QQ音乐检索与爬取工具

一个基于 Streamlit 的可视化 QQ 音乐数据抓取工具，支持：

- 先检索再爬取：按歌手、按歌名
- 按歌手热度抓取前 N 首歌曲
- 按歌手先检索 -> 获取该歌手歌曲列表 -> 再决定抓取前 N 首或手动勾选
- 抓取歌曲详情字段（歌名、专辑、语种、流派、唱片公司、发行时间、简介、歌词、评论数）
- 抓取评论明细（点赞数、评论人名、时间、地点、内容）
- SQLite 分表存储，并在 UI 中展示与导出
- 按评论点赞数排序与高赞过滤
- 一键导出 Excel（多 sheet：songs/comments）
- 多线程抓取 + 请求重试 + 随机延时

## 快速开始

1. 安装依赖

```bash
pip install -r requirements.txt
```

2. 运行界面

```bash
streamlit run app.py
```

或直接双击：`启动QQ音乐爬取工具.bat`

3. 打开浏览器访问提示的本地地址（通常是 `http://localhost:8501`）

## 数据表设计

- `songs`：歌曲主表（每首歌一条）
- `comments`：评论明细表（每条评论一条）

## 注意事项

- 本项目使用公开接口进行学习与课程实验用途。
- 接口策略可能随时间变化，部分字段可能为空。
- 请遵守目标平台服务条款与相关法律法规。

## 并发与稳定性参数

在“按歌手爬取/按歌名爬取”页面中的“可选高级设置（稳定性/速度）”里可配置：

- 评论抓取策略：
- 最新评论按“加载批次 + 每批条数”控制
- 近期热评、精彩评论可通过复选框附加抓取（并自动去重合并）
- 并发线程数
- 请求超时、最大重试次数、重试退避系数
- 随机延时最小值/最大值
